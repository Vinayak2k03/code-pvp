## Data Flow

### 1. Authentication Flow
```
Client вҶ’ POST /api/auth/signup вҶ’ Hash password вҶ’ Store in DB вҶ’ Return JWT
Client вҶ’ POST /api/auth/login  вҶ’ Verify password вҶ’ Return JWT
Client вҶ’ All requests include Authorization: Bearer <token>
```

### 2. Matchmaking Flow
```
Client вҶ’ Socket: "queue:join" вҶ’ Add to Redis sorted set (by rating)
Redis вҶ’ Every 2s: scan queue вҶ’ Find Вұ100 rating match
Match found вҶ’ Create Match in DB вҶ’ Create Socket Room
Socket вҶ’ Emit "match:found" to both players вҶ’ Navigate to /duel/:matchId
```

### 3. Code Execution Flow
```
Client вҶ’ Socket: "code:submit" вҶ’ Server validates
Server вҶ’ POST Judge0 /submissions (batch for all test cases)
Server вҶ’ Poll Judge0 for results (or use callback)
Server вҶ’ Evaluate: all passed = Accepted, else WA/TLE/RE
Server вҶ’ Socket: "submission:result" to player
Server вҶ’ Socket: "opponent:submitted" to opponent
If Accepted вҶ’ Check if first solver вҶ’ End match вҶ’ Update ELO
```

### 4. ELO Rating System
```
K-factor: 32 (default), 16 (rating > 2000)
Expected score: E = 1 / (1 + 10^((Rb - Ra) / 400))
New rating: Ra' = Ra + K * (S - E)
  where S = 1 (win), 0.5 (draw), 0 (loss)
```

### 5. WebSocket Events

| Event | Direction | Payload | Description |
|-------|-----------|---------|-------------|
| `queue:join` | CвҶ’S | `{ userId }` | Join matchmaking queue |
| `queue:leave` | CвҶ’S | `{ userId }` | Leave queue |
| `queue:status` | SвҶ’C | `{ position, estimated }` | Queue position update |
| `match:found` | SвҶ’C | `{ matchId, opponent, problem }` | Match created |
| `match:ready` | CвҶ’S | `{ matchId }` | Player ready |
| `match:start` | SвҶ’C | `{ matchId, startTime }` | Match begins |
| `code:run` | CвҶ’S | `{ matchId, code, language }` | Run code |
| `code:submit` | CвҶ’S | `{ matchId, code, language }` | Submit solution |
| `code:result` | SвҶ’C | `{ verdict, time, memory }` | Execution result |
| `opponent:status` | SвҶ’C | `{ status }` | Opponent activity |
| `match:end` | SвҶ’C | `{ winner, ratings }` | Match ended |
| `spectator:join` | CвҶ’S | `{ matchId }` | Join as spectator |
| `spectator:update` | SвҶ’C | `{ submissions }` | Live updates |

## API Routes

### Auth
- `POST /api/auth/signup` вҖ” Register
- `POST /api/auth/login` вҖ” Login
- `GET /api/auth/me` вҖ” Current user

### Users
- `GET /api/users/:id` вҖ” User profile
- `GET /api/users/:id/stats` вҖ” User statistics

### Problems
- `GET /api/problems` вҖ” List problems (paginated, filterable)
- `GET /api/problems/:id` вҖ” Problem detail (public test cases only)

### Matches
- `GET /api/matches` вҖ” Match history (authenticated user)
- `GET /api/matches/:id` вҖ” Match detail
- `GET /api/matches/live` вҖ” Ongoing matches (for spectators)

### Leaderboard
- `GET /api/leaderboard` вҖ” Global rankings (paginated)

### Submissions
- `GET /api/submissions/:matchId` вҖ” Submissions for a match
